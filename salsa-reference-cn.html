Create User API Token · 创建用户 API Token

POST /api/rest/v1/auth/token

用户 API token 为一次性操作提供短期有效、权限受限的身份认证凭证。请求体必须是以下两种类型之一(通过 type 字段区分):雇主用户 token 或员工用户 token。

请求体(类型一):CreateEmployerUserTokenInput

字段类型必填说明
typestring(固定 CreateEmployerUserTokenInput)是请求体类型标识
employerIdsstring[]是雇主 ID 数组(至少 1 个,单个 token 最多 125 个雇主)
roleenum是赋予该 token 的权限组。可选值:EMPLOYER_BASIC / EMPLOYER_ADMIN / EMPLOYER_SUPER_ADMIN / EMPLOYER_ONBOARDING_BASIC / EMPLOYER_ONBOARDING_ADMIN / EMPLOYER_ONBOARDING_SUPER_ADMIN
timeToLiveInMinutesinteger否token 有效时长(分钟),默认 60,最小 1
userIdentifierstring否你在外部平台中该用户的唯一引用 ID(可选,但建议提供)

请求体(类型二):CreateWorkerUserTokenInput

字段类型必填说明
typestring(固定 CreateWorkerUserTokenInput)是请求体类型标识
workerIdstring是token 归属的员工 ID(长度至少 1)
roleenum是赋予该 token 的权限组。可选值:WORKER_BASIC / WORKER_ADMIN / WORKER_SUPER_ADMIN / WORKER_ONBOARDING_BASIC / WORKER_ONBOARDING_ADMIN / WORKER_ONBOARDING_SUPER_ADMIN
timeToLiveInMinutesinteger否token 有效时长(分钟),默认 60,最小 1
userIdentifierstring否你在外部平台中该用户的唯一引用 ID

响应字段(RestApiEnvelopeApiToken)

字段类型必填说明
tokenstring是发起请求所需的唯一认证密钥
expirationstring(date-time)是token 过期时间戳(ISO 8601 格式)

错误响应:400/404/422 返回 ErrorApiEnvelope,500 服务器内部错误。认证方式:Bearer token。服务器:沙箱 https://api.sandbox.salsa.dev,生产 https://api.salsa.dev。

Retrieve Document · 获取单个文档

GET /api/rest/v1/documents/{documentId}

根据指定的文档 ID 获取对应的 Document(文档)对象。

路径参数

字段类型必填说明
documentIdstring是文档 ID

响应字段(Document)

字段类型必填说明
idstring是文档的唯一标识
namestring是文件名
categoryDocumentCategory是文件所属分类
statusDocumentStatus是标识文件是否已渲染完成、可供使用
createdAtstring(date-time)是文档创建时间
descriptionstring | null否文件描述
mediaTypestring | null否文件的媒体类型,遵循 RFC 2046(示例:application/pdf)
documentLinksarray是用于下载或上传该文件的文档链接列表

枚举值

枚举可选值
DocumentCategoryWORKER_EARNINGS_RECORD(员工收入记录)、WORKER_PAYCHECK(员工薪资单)、TAXES(税务)、SETUP(设置)、REPORTS(报表)、OTHER(其他)

错误响应:400/404/422 返回 ErrorApiEnvelope,500 服务器内部错误。认证:Bearer token。

Send Paystream Item · 发送 Paystream 数据项（通用入口）

通过 Paystream 接入各类薪酬相关数据，涵盖：Employers（雇主）、Workers（员工/劳动者）、Pay（薪资）、Time Worked（工时）、Time Off（休假）、Benefits（福利）、Deductions（扣款）。

默认情况下各数据项异步处理；将查询参数 synchronous=true 设为真则内联同步处理。同步响应中每一项会带上 processed 字段（含 Salsa 生成的 ID），异步响应的 processed 为 null。

字段类型必填说明
dataarray是待 Paystream 处理的输入数据数组

data 数组中每一项可为以下几种类型之一（通过 type 判别字段区分）：

  • PaystreamEmployerUpsertInput（Upsert Employer Input，创建/更新雇主）
  • PaystreamWorkerUpsertInput（Upsert Worker Input，创建/更新员工）
  • PaystreamWorkerCreateInput（Create Worker Input，已弃用）
  • PaystreamWorkerUpdateInput（Update Worker Input，更新员工）
  • PaystreamPayrollElementPeriodReplacementInput（Replace Period Input，替换某周期内的薪资要素）
  • PaystreamEmployerRiskMetricsOnPartnerPlatformUpsertInput（合作平台上的雇主风险指标）

各类型的具体字段见下文对应端点章节（Import Employers / Import Workers / Import Payroll Elements / Import employer risk metrics）。同步处理时，每项返回的 processed 形状与各专用端点一致。

Salsa.js (Web SDK) · Salsa.js 浏览器端 SDK

用途:Salsa 的浏览器端 JavaScript 库,叫 Salsa.js。它提供一组模块化 UI 组件(称为 "Salsa Elements"),让你把 payroll 相关功能无缝嵌入到自己的 Web 应用里。上面各个组件(worker-profile、employer-reports 等)都是通过这个 SDK 来创建和挂载的。

安装 / 加载(Installation)

方式一:CDN Script 标签(推荐)

<script type="text/javascript" src="https://js.salsa.dev/v0"></script>

加载后,通过 window 对象上的 Salsa 函数访问该库。可以给 script 加上 async 或 defer 属性实现异步加载。

方式二:NPM 包

npm install @salsa-payroll/salsa-js
import { loadSalsaJs } from "@salsa-payroll/salsa-js";

const salsa = await loadSalsaJs('ck_your-client-key');

初始化 Salsa.js:Salsa(clientKey, options?)

创建一个库实例——这是使用整个 SDK 的入口。每个应用运行时只需初始化一次。

名称类型说明
clientKeyString(必填)你的应用的 Client Key(格式形如 ck_f0ac...eb6d8),用于让 Salsa 识别你的应用。
options.env'sandbox' | 'production'选择环境;不传时默认为 production(生产环境)。

创建组件:salsa.elements.create(elementType, options?)

创建一个具体的 Salsa Element(组件)实例。

名称类型说明
elementTypeString(必填)要创建的 Element 类型(如 worker-profile)。
userTokenString用某个具备足够权限的访问角色创建出来的 User Token(几乎所有 element 都需要它)。
style.colorMode'light' | 'dark' | 'system'组件的颜色模式;默认为 light。'system' 会跟随浏览器/操作系统的设置。
allowRequestForPrivilegedAccessBoolean是否启用"身份核验升权(step-up)"流程;默认为 false。

组件实例方法(Element Instance Methods)

挂载组件:salsa.elements.mount(salsaElement, domElement)——把一个 SalsaElement 挂载到 DOM 中 domElement 指定的位置。

名称类型说明
salsaElementSalsaElement(必填)要挂载的组件实例。
domElementString(必填)挂载位置的 CSS 选择器或 DOM 元素。

销毁组件:salsa.elements.destroy(salsaElement)——把组件从 DOM 移除,清理句柄并注销监听器。

名称类型说明
salsaElementSalsaElement(必填)要销毁的组件实例。

更新鉴权 token:salsaElement.replaceUserToken(newUserToken)——在不刷新组件的情况下更换 user token,适用于 token 即将过期、或需要在会话中途调整访问级别的场景。(文档给出的示例展示了如何用 setInterval 在 token TTL 到期前自动刷新。)

名称类型说明
newUserTokenString(必填)新的 user 鉴权 token。

取消升权请求:salsaElement.cancelRequestForPrivilegedAccess()——关闭由"申请特权访问"触发的核验弹窗,把用户带回上一个视图,不改变 token。无参数、无返回值;若当前没有待处理的升权请求则无任何效果。

事件系统(Event System)

注册事件监听:<SalsaElement>.on(eventName, handler)——按原生 web 元素的方式注册事件监听器。

名称类型说明
eventNameString(必填)要监听的事件名。
handlerFunction(必填)事件触发时调用的回调 handler(event) => void。

可以对同一个 eventName 多次调用 on 来挂载多个监听器。

complete 事件:某个流程完成时触发,可用来据此更新 UI。事件对象属性:

名称类型说明
elementTypeString发出该事件的 Element 类型。
(其他属性)varies随组件不同而不同的专属数据。

request-privileged-access 事件:当用户尝试执行超出其当前角色权限的操作时触发(仅在创建组件时设置了 allowRequestForPrivilegedAccess: true 才会有)。它是"身份核验升权(step-up)"流程的入口。事件对象属性:

名称类型说明
actionString被保护操作的标识符(见下方 Action IDs 表)。
recommendedRoleString(选填)满足该操作所需的"最低角色"(以 createUserToken 的格式表示,如 EMPLOYER_SUPER_ADMIN);当用户无法升权时会省略。
possibleRolesString[](选填)所有能满足该操作的角色,按权限高低排序;与 recommendedRole 一同省略。

Action IDs(被保护操作标识符):

名称说明
download-document打开/下载员工或雇主的文档。
add-employer-bank-account新增雇主银行账户。
add-worker-bank-account新增员工银行账户。
delete-worker-bank-account删除员工银行账户。
unmask-employer-bank-account-number显示雇主银行账户的完整账号(取消打码)。
unmask-worker-bank-account-number显示员工银行账户的完整账号(取消打码)。
unmask-government-id显示员工的完整政府证件号(SSN/TIN,取消打码)。
update-worker-government-id修改员工的政府证件号。
download-worker-details-report-with-government-ids下载含未打码证件号的 Worker Details 报表。
